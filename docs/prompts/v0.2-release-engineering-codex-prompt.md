# Sub-prompt 4: Makefile local parity

## Purpose

Add Makefile targets that mirror local and CI verification paths.

## Required target names

Use short conventional target names:

```text
install
lint
test
eval
check
verify
compose-check
compose-build
ci-local
```

## Required semantics

```makefile
COMPOSE ?= docker compose -f infra/compose.yaml
```

Target meanings:

- `make install`
  - runs `uv sync --group dev`

- `make lint`
  - runs Ruff lint only
  - expected command: `uv run ruff check .`

- `make test`
  - runs normal pytest suite only
  - does not run deterministic evals

- `make eval`
  - runs the existing deterministic eval mechanism
  - do not invent placeholder eval behaviour
  - if eval command is unclear, stop and report

- `make check`
  - runs lint + normal tests

- `make verify`
  - runs lint + normal tests + deterministic evals

- `make compose-check`
  - validates Compose configuration with `$(COMPOSE) config --quiet`

- `make compose-build`
  - builds Compose images with `$(COMPOSE) build`

- `make ci-local`
  - runs full CI parity
  - includes `make verify`
  - includes Compose config validation
  - includes Compose image build
  - fails if Docker/Compose is unavailable

Docker is the primary path. Podman is best-effort via override:

```bash
make compose-check COMPOSE="podman compose -f infra/compose.yaml"
make compose-build COMPOSE="podman compose -f infra/compose.yaml"
```

## Verification

Run:

```bash
make check
make verify
```

Run if Docker/Compose is available:

```bash
make compose-check
make compose-build
make ci-local
```

## Stop-and-report checkpoint

Report:

- Makefile targets added
- exact test command
- exact eval command
- exact Compose command
- commands run
- pass/fail results
- skipped checks and why
- risks/issues
- recommended next sub-prompt

Stop after reporting.
